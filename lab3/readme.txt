****************************************************************************************************

В результате работы программы генерируются следующие результаты в папке proc:
em_iters.png - сеймплы из распределений каждого из классов на каждой итерации
               ЕМ-алгоритма, в конце добавлен сеймпл из распределения, которое
               было получено в 1 лабораторной работе из прямоугольников
start.png   - классификация с использованием распределений, полученных ЕМ
final.png   - классификация после применения итераций семплера Гиббса
diff.png    - показывает, какие пиксели изменили свой класс из-за Гиббса
iter/i.png  - классификация и выделение изменений на i-ой итерации Гиббса
probs/i.png - соотношение распределений в каждом пикселе на i-ой итер. Г.

****************************************************************************************************

По поводу структуры em_iters.png поподробнее:
[ Номер итерации ]
------------------------------------------------------------------------------
[ Сеймпл (сгенерированные значения) используя текущее распределение 0 класса ]
[ Сеймпл (сгенерированные значения) используя текущее распределение 1 класса ]
------------------------------------------------------------------------------
[ Значение, которое максимизируется (argmax) в пункте 2c для 0 класса ]
[ Значение, которое максимизируется (argmax) в пункте 2c для 1 класса ]
[ Сумма выше указанных значений ]

****************************************************************************************************

Начальные распределения для ЕМ-алгоритма специально выбраны не самым лучшим образом,
чтобы можно было буквально увидеть эволюцию распределений. Если брать, например, крайние
распределения (со средними 0 и 255) то сходимость происходит за пару итераций.
